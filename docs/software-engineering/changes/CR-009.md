## Request

Memperbarui tampilan Cloudflare agar lebih modern dan rapi, menambahkan dark mode, menyusun dashboard sesuai aktor, menambahkan gerbang Google Login yang dilanjutkan pemilihan role demo, lalu mempublikasikan link aktif melalui GitHub.

## Approved Scope

- Modern responsive shell, dark/light mode, and clearer dashboard per role.
- Login landing screen with Google OAuth endpoint and a clearly labeled academic demo fallback.
- Role selection remains the established demo mechanism from REQ-022; core lifecycle remains unchanged.
- Google credentials remain Cloudflare secrets and must never enter Git.
- Deploy accepted candidate and update GitHub documentation.

## Not In Scope

Production identity provisioning, organization allow-list, permanent Google-user-to-role assignments, new D1 tables, or replacing the academic role-selector authorization model.

## Before

Users enter directly through a header role selector. Styling is light-only and all actors share nearly the same page organization.

## After

Users see a polished sign-in surface, may use configured Google OAuth or an explicit demo entry, select a demo role, and receive a role-oriented workspace with persistent dark mode. Workflow transitions remain REL-002 compatible.

## Verification Needed

Unit tests, lint, production build, keyboard/accessibility inspection, role visibility checks, OAuth unconfigured-state check, Cloudflare smoke tests, and GitHub README link verification.

## Rollback and Compatibility

Rollback to REL-002 Worker version `fb4c4545-7f09-4560-bca4-fe2598780926`. No migration or D1 rollback is needed. If OAuth secrets are absent, demo login remains available and the Google route returns a controlled configuration error.

## Baseline Update Rule

Promote REL-003 only after tests, UAT, deploy, and smoke verification pass. Google OAuth is marked configured only after all three secrets exist and a real callback succeeds.

## Routing Decision

Route to UI design, issue planning, implementation, review, automated testing, acceptance testing, deployment, and change-control closure.
