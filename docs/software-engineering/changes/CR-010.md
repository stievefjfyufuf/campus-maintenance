Pindahkan pemilihan role ke halaman login sebelum tombol Google, bawa pilihan tersebut melalui OAuth hingga pengguna masuk ke workspace yang benar, pastikan logout berfungsi, verifikasi backend tanpa error, dan pertahankan URL Cloudflare untuk masa penilaian sekitar satu bulan.

## Approved Scope

- Select one of PELAPOR, ADMIN, TEKNISI, or MANAJER before login.
- Preserve selected role through Google OAuth and demo entry.
- Remove in-app role switching; changing role requires logout and login again.
- Verify logout, API health, D1 reads, tests, lint, build, deploy, and production smoke.
- Keep the public URL in README.

## Not In Scope

Creating a Google Cloud project on the user's behalf, fabricating OAuth credentials, guaranteeing third-party uptime, or replacing the academic role model with production RBAC.

## Before

The role selector appears in the application header after login/demo entry. OAuth state does not preserve a chosen role.

## After

The user selects a role on the login page, then chooses Google or demo login. The selected role is validated and carried into the resulting session. Logout returns to the login screen and clears the Google session cookie.

## Verification Needed

26 regression tests, lint, build, OAuth configuration endpoint, invalid-role fallback, demo role entry, logout/session clearing, API health, D1 reference reads, deploy, and production smoke.
