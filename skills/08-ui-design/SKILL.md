# UI Design

## Purpose

Gunakan skill ini untuk mengubah user story, acceptance criteria, dan API contract menjadi rancangan UI yang siap diimplementasikan.

## Required Inputs

- `docs/software-engineering/03-specification.md`
- `docs/software-engineering/04-prioritization.md`
- `docs/software-engineering/07-database-api-design.md`

## Workflow

1. Turunkan screen dari user stories dan role.
2. Buat navigation flow dan primary user journey.
3. Rancang form fields, validation messages, loading/empty/error/success states.
4. Pastikan permission dan data visibility mengikuti REQ/AC/API.
5. Sertakan wireframe text atau diagram sederhana.

## Output

Simpan artefak ke `docs/software-engineering/08-ui-design.md`.

## Handoff

Kirim screen specs dan flow ke Step 9 issue planning.
