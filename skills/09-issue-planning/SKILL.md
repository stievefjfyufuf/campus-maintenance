# Issue Planning

## Purpose

Gunakan skill ini untuk memecah scope tervalidasi menjadi issue implementasi yang kecil, traceable, dan dapat direview.

## Required Inputs

- `docs/software-engineering/03-specification.md`
- `docs/software-engineering/04-prioritization.md`
- `docs/software-engineering/05-validation-change.md`
- `docs/software-engineering/06-architecture-design.md`
- `docs/software-engineering/07-database-api-design.md`
- `docs/software-engineering/08-ui-design.md`

## Workflow

1. Buat satu issue untuk satu outcome reviewable.
2. Catat context, scope, acceptance checklist, technical notes, dependencies, dan out-of-scope.
3. Urutkan issue berdasarkan dependency dan prioritas.
4. Hubungkan setiap issue ke REQ, AC, ADR, DATA, API, UI, dan CR bila ada.

## Output

Simpan artefak ke `docs/software-engineering/09-issue-planning.md`.

## Handoff

Kirim issue terpilih ke Step 10 implementation.
