# Acceptance Tests

Alur UAT dan hasil penerimaan dicatat di `../../docs/software-engineering/14-acceptance-testing.md`.

Alur demo minimum:

1. Pelapor membuat laporan.
2. Administrator memulai review dan menugaskan teknisi.
3. Teknisi menerima tugas dan menandai pekerjaan selesai.
4. Pelapor atau administrator menutup laporan.
5. Administrator/manajer melihat metrik dashboard; pelapor/teknisi tidak melihat panel dashboard.

Gunakan data sintetis saat demo dan jangan memasukkan kontak pribadi nyata.
